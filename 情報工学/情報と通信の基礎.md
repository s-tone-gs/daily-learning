# 情報と通信の基礎

- カテゴリ: 情報工学
- 最終更新: 2026-06-20

[情報セキュリティの基本](../セキュリティ/情報セキュリティの基本.md)で出てくる「情報通信の特徴」は、ここに書く前提知識の上に成り立っている。

---

## 1. 情報が伝わるには、送信側と受信側が同じ「規約」を共有している必要がある

電気信号そのものは「猫がいる」という意味を持っていない。ただの電圧の変化(物理現象)に過ぎない。受け取った側が「この電圧パターンはこの文字に対応する」という**規約(符号化・復号化のルール)**を知っていて初めて、元の意味に戻せる。

声(音波)も同じ。日本語を知らない人にとっては、「猫がいる」という音はただの音であり、情報にならない。日本語という規約を共有している人だけが、その音から意味を取り出せる。

つまり、「猫がいる」という情報が、声で話しても、紙に書いても、電気信号にしても同じ情報として伝わるのは、**情報そのものが媒体と無関係に存在しているから、ではない**。声も文字も電気信号も、最終的に同じ規約(日本語という言語、文字コードなど)に対応づけられていて、送信側と受信側がその規約を共有しているからこそ、同じ情報として伝わる。規約を共有していなければ、同じ物理現象を受け取っても情報としては伝わらない。

この「規約の共有」という考え方は、暗号化の仕組みとも直結している。普通の通信は規約(言語、文字コードなど)をできるだけ多くの人と共有することで、誰にでも情報が伝わるようにしている。暗号化された通信は、規約(鍵)を意図的に特定の人だけに共有することで、その人以外には情報が伝わらないようにしている。暗号化とは、「情報が伝わるには規約の共有が必要」という性質を逆手に取り、規約を絞ることで伝わる相手を制限する技術だと言える。

## 2. どんな情報も、記号の並びとして表現できる

声、文字、画像、音楽、どんな種類の情報でも、最終的には「記号の並び」として書き出せる。これは20世紀半ばにクロード・シャノンが情報理論として整理した内容で、「情報の正体は、いくつかの選択肢の中からどれが選ばれたか、という選択の積み重ねである」という捉え方をする。

## 3. なぜ「0と1」(2種類)が選ばれたのか

記号の種類は2つでなくても理論上は成立する(10進数でも、もっと多い種類の記号でも可能)。しかし2種類(電圧が高いか低いか、光があるか無いか、磁気の向きがどちらか)を区別する仕組みは、3種類以上を正確に区別する仕組みよりも圧倒的に作りやすく、ノイズ(電気的な揺れや劣化)があっても誤読しにくい。

- 例: 「高い/低い」の2択なら、多少の電圧のブレがあっても「だいたいどちらか」は判定しやすい
- 「10段階の電圧」を正確に区別しようとすると、わずかなブレで読み間違いが起きやすくなる

この「区別のしやすさ・壊れにくさ」が、コンピュータや通信のほぼ全てで2進数(バイナリ)が採用されている理由。

## 4. 記号を、実際の物理現象に対応させる

「0」「1」という抽象的な記号を、実際に存在する物理現象(電圧の高低、光の点滅、電波の振幅や周波数の違いなど)に対応させることで、初めて記号を実際に送ったり保存したりできるようになる。これを「変調」「符号化」と呼ぶ。

## 5. 通信とは、その物理現象を送信側から受信側まで伝えること

信号(電気、光、電波)を、ケーブルや空間といった「媒体」を通して、送信側の場所から受信側の場所まで届けること。これが通信の定義。

## まとめ:ここから情報セキュリティの話につながる

```
情報が伝わるには、送信側と受信側が同じ規約を共有している必要がある(1)
  → どんな情報も記号の並みに変換できる、というのも1つの規約(2)
    → 2種類の記号(0/1)が最も扱いやすい(3)
      → 記号を物理現象に対応させる(4)
        → その物理現象を媒体を通して送る、これが通信(5)
          ├─ 物理現象(電波・電気信号)は特定の人だけのものにできない → 占有できない
          ├─ 記号は劣化なくそのまま複製できる → コピーと元データが見分けつかない
          ├─ 記号の書き換えは別の正常な記号に見えるだけ → 改ざんの跡が残らない
          ├─ 記号は形式的なので数式(関数)で変換できる → 数式で処理できる
          └─ 信号の伝達・処理は電気・光の速さで動く → 機械の速さで動く
```

「情報が伝わるには規約の共有が必要」という根源的な前提から、「規約の一種として記号化(0/1)が使われている」という流れで、最終的に[情報セキュリティの基本](../セキュリティ/情報セキュリティの基本.md)で扱う5つの特徴に行き着く。同じ「規約の共有」という考え方は、[情報セキュリティの基本](../セキュリティ/情報セキュリティの基本.md)で扱う暗号化(規約=鍵を特定の人だけに共有する技術)にも直結している。
